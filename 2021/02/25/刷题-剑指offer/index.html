03.数组中重复的数字(Easy)

问题：

在一个长度为 n 的数组 nums 里的所有数字都在 0～n-1 的范围内。数组中某些数字是重复的，但不知道有几个数字重复了，也不知道每个数字重复了几次。请找出数组中任意一个重复的数字。

思路：

数组nums长度为n，数字范围0~n-1。

若没有重复数字，排完序后nums[i]值一定是i。

于是，在循环中，对数组下标为i的nums[i]，令与nums[nums[i]]上的位置交换，每一次交换，能确定nums[nums[i]]上的值就等于其下标。

在交换完毕后，若nums[i]与nums[nums[i]]相等，但二者位置不同，则存在重复数字。

代码：

class Solution {
    public int findRepeatNumber(int[] nums) {
        for(int i = 0; i < nums.length; i++){
            //比较，不同则交换
            while(nums[i] != nums[nums[i]]){
                int t = nums[nums[i]];
                nums[nums[i]] = nums[i];
                nums[i] = t;
            }
            if(nums[i] == nums[nums[i]] && i != nums[i]){
                return nums[i];
            }
        }
        return -1;
    }
}

复杂度：

时间复杂度：O(n)

空间复杂度：O(1)

04.二维数组中的查找(Medium)

问题：

在一个n*m 的二维数组中，每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个高效的函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

示例：

现有矩阵matrix如下：

[
  [1,   4,  7, 11, 15],
  [2,   5,  8, 12, 19],
  [3,   6,  9, 16, 22],
  [10, 13, 14, 17, 24],
  [18, 21, 23, 26, 30]
]

给定target = 5，返回true

给定target = 20，返回false

思路：

观察示例中的矩阵，注意到，由右上角元素开始，若当前元素大于目标值，则自元素以下的一整列均大于目标值，必须向左才能找到目标。若当前元素小于目标值，则自元素往左的一整行均小于目标值，必须往下才能找到目标。

在我找到的一份题解里画了一份很棒的图，把这个思路描述得十分清楚，见下。

代码：

class Solution {
    public boolean findNumberIn2DArray(int[][] matrix, int target) {
        int n = matrix.length;
        if(n == 0){
            return false;
        }
        int m = matrix[0].length;
        int i = 0;
        int j = m-1;
        while(i < n &&  j >= 0){
            if(matrix[i][j] == target){
                return true;
            }
            else if(matrix[i][j] > target){
                j--;
            }
            else{
                i++;
            }
        }
        return false;
    }
}

复杂度：

时间复杂度：O(n+m)

空间复杂度：O(1)
